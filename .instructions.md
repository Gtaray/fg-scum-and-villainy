# Scum and Villainy Ruleset — Project Instructions

This is a Fantasy Grounds Unity ruleset for **Scum & Villainy** (by John Harper / One Seven Design), a Forged in the Dark space opera TTRPG. The ruleset layers on CoreRPG.

**Author**: Saagael

## The Game System: Forged in the Dark

### Core Mechanic
Players roll pools of d6s (action rolls). The highest die determines the outcome:
- **6**: Full success
- **4-5**: Partial success (with consequence)
- **1-3**: Bad outcome
- **Critical** (multiple 6s): Enhanced success

Position (Controlled/Risky/Desperate) and Effect (Limited/Standard/Great) frame every roll.

### The Three Attributes & Twelve Actions
Each attribute contains 4 actions. Action ratings (0-4 dots) determine dice pools. Attribute score = number of actions with rating > 0.

| Attribute | Actions | Used for resistance against... |
|-----------|---------|-------------------------------|
| **Insight** | Doctor, Hack, Rig, Study | Consequences from deception or understanding |
| **Prowess** | Helm, Scramble, Scrap, Skulk | Physical consequences |
| **Resolve** | Attune, Command, Consort, Sway | Mental/social consequences |

Note: In Blades in the Dark, the actions differ (Survey instead of Hack, Tinker instead of Rig, Finesse instead of Helm, Prowl instead of Skulk, Hunt instead of Scrap, Wreck instead of Scramble). The attributes remain the same.

### Stress & Trauma
- **Stress** (0-9): Resource spent to push rolls, resist consequences, activate abilities
- **Trauma** (0-4): Gained when stress maxes out. Permanent conditions (Cold, Haunted, Obsessed, Paranoid, Reckless, Soft, Unstable, Vicious)
- Filling trauma = character retirement

### Harm (3 levels)
- Level 3 (Severe): 1 slot, -1d to all actions needing that ability
- Level 2 (Moderate): 2 slots, -1d penalty
- Level 1 (Lesser): 2 slots, less severe penalty
- Recovery tracked via a progress clock

### Resistance Rolls
When suffering a consequence, a player may resist using the relevant attribute. Roll attribute score in d6s. Stress cost = 6 minus highest die (critical = 0 stress, gain +1d on related action).

### Vice Roll
Roll the PC's lowest attribute score. Result determines how much stress is cleared during downtime.

### Loadout
Characters choose load level before a score:
- Light (3 slots)
- Normal (5 slots)  
- Heavy (6 slots)

Individual items cost 1-2 load each. Items are marked as equipped during play.

### Clocks
Progress clocks (4/6/8/10/12 segments) track complex obstacles, dangers, faction projects, and long-term progress. Used everywhere: scores, downtime, faction activities, crew advancement.

### Crew & Ship (Party Sheet)
The crew shares:
- **Crew type** (with special abilities and XP triggers)
- **Ship** with sections: Hull, Engines, Weapons, Comms (each rated)
- **Reputation** (1-3 rating, with XP track)
- **Heat** per star system (0-8), **Wanted level** per system (0-4)
- **Cred** (spending money) and **Debt**
- **Gambits** (crew resource for bonus dice)
- **Upkeep** = floor((crew rating + sum of section ratings) / 4)
- **Factions** with status ranging from -3 (war) to +3 (allied)

### Heat & Wanted
Heat is tracked **per star system** (not globally). When heat fills (8), it clears and the wanted clock advances. When the wanted clock fills, wanted level increases (max 4).

### Engagement Roll
At the start of a score/job, the GM makes an engagement roll to determine starting position. Uses a variable dice pool based on the plan type (Assault, Deception, Infiltration, Mystic, Social, Transport) plus situational factors.

### Advancement
- **Playbook XP** (max 8): Earned by expressing beliefs/drives/heritage/background and desperate actions
- **Attribute XP** (max 6 per attribute): Earned by using actions of that attribute in risky/desperate situations
- **Crew XP** (max 8): Earned by completing jobs matching crew type

### Deep Cuts (Supplement)
An optional supplement that overhauls:
- Advancement rules (the "Deep Cuts - Advancement" extension)
- Loadout rules (the "Deep Cuts - Loadout" extension)
- Stress/Harm rules (the "Deep Cuts - Stress and Harm" extension)
- Threat rolls (the "Deep Cuts - Threat Rolls" extension)

## Ruleset Architecture

### Global Scripts
| Script Name | File | Purpose |
|---|---|---|
| `GameSystem` | `scripts/manager_gamesystem.lua` | Roll action types, record type overrides, sidebar categories |
| `DataManager` | `scripts/manager_data.lua` | Core game data definitions (attributes, actions, ship sections, sizes, traumas, plans) |
| `CharManager` | `scripts/manager_char.lua` | Character operations (scores, stress, harm, gear, abilities) |
| `CrewManager` | `scripts/manager_crew.lua` | Crew/party sheet data, OOB messaging for player edits |
| `JobManager` | `scripts/manager_job.lua` | Job/Score management (replaces combat tracker) |
| `ClockManager` | `scripts/manager_clock.lua` | Clock sizes, creation/drop logic, chat messaging |
| `RollManager` | `scripts/manager_roll.lua` | Position/Effect, dice pool building, result processing |
| `ActionDice` | `scripts/manager_action_dice.lua` | Generic d6 pool roll handler |
| `ActionRoll` | `scripts/manager_action_roll.lua` | Action roll (push/bargain/assist/gambit/injury mods) |
| `ActionEngagement` | `scripts/manager_action_engagement.lua` | Engagement roll → starting position |
| `ActionFortune` | `scripts/manager_action_fortune.lua` | Fortune roll (no position/effect) |
| `ActionResistance` | `scripts/manager_action_resistance.lua` | Resistance roll → stress cost |
| `ActionVice` | `scripts/manager_action_vice.lua` | Vice roll (lowest attribute score) |

### Character Sheet Database Schema
```
charsheet.[id]/
├── playbook (link, name, tagline, xp max=8)
├── heritage (name, text)
├── background (name, text)
├── health/
│   ├── stress (current, max=9)
│   ├── trauma (current, max=4)
│   ├── harm (severe, moderate1, moderate2, lesser1, lesser2)
│   ├── recovery (progress, clocksize=6)
│   └── vice (bonusdice)
├── defenses/ (armor, heavy, special — each with spent/max)
├── attributes/[child]/
│   ├── attribute (string key)
│   ├── resistance.bonusdice
│   ├── xp (current, maximum=6)
│   └── actions/[child]/ (action key, rating 0-4, bonusdice)
├── inventory/
│   ├── load (light/normal/heavy flags, amounts: 3/5/6)
│   └── gear/[child]/ (name, load, equipped, type, shortcut)
├── abilities/[child]/ (link, paid, cost)
├── clocks/[child]/ (name, size, progress, notes)
├── contacts/[child]/ (name, notes, relationship)
└── advancement/ (xptriggers list)
```

### Crew Sheet Database Schema (`partysheet` node)
```
partysheet/
├── crew/ (name, reputation, rating max=3, contacts)
├── ship/ (link, class, size, look)
├── sections/[child]/ (name, rating, maxrating, upgrades)
├── upgrades/ (specialabilities, crewupgrades, auxiliary, shipgear, crewgear, training)
├── factions/[child]/ (link, status -3 to +3)
├── systems/[child]/ (link, heat max=8, wanted max=4, wantedclock)
├── advancement/ (xp max=8, xptriggers)
├── cred/debt (current, max)
├── upkeep (total, skips)
├── gambits (current, default, max)
└── clocks/[child]/ (name, size, progress, notes, visible)
```

### Job Sheet (`js/` folder)
Replaces the combat tracker. Manages active scores with engagement roll results, notes, and clocks.
